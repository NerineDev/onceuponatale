# Roadmap

This roadmap outlines the current high-level direction for **Once Upon a Tale**.

It is intentionally broad and spoiler-light. Specific narrative structures, route conditions, and unreleased story content are being kept private during development.

## Development Phases

### Phase 1 — Foundation

- Define the core premise and narrative identity of the project
- Establish tone, themes, and overall direction
- Develop early worldbuilding structure
- Explore visual direction and presentation style
- Organize internal documentation and planning pipelines

**Status:** In progress

---

### Phase 2 — Narrative Structure

- Build the general story framework
- Define route architecture
- Plan progression flow and route access logic
- Develop major character concepts and relationship dynamics
- Identify core emotional beats and route differentiation
- Structure common-route and character-route foundations

**Status:** In progress

---

### Phase 3 — Systems Design

- Plan branching dialogue logic
- Define stat systems and progression rules
- Outline condition-based unlocks
- Design minigame integration at a high level
- Map save/load and player progression needs
- Establish internal naming conventions and logic structure

**Status:** In progress

---

### Phase 4 — UI / UX Prototype

- Create interface direction for menus and in-game screens
- Prototype narrative presentation flow
- Explore visual hierarchy, readability, and mood
- Build placeholder layouts for key screens
- Refine the overall player experience from first launch onward

**Status:** Planned / ongoing exploration

---

### Phase 5 — Art Pipeline

- Establish character design workflow
- Refine style consistency for sprites and illustrations
- Plan event artwork / CG needs
- Develop placeholder asset list
- Build a production pipeline for future art implementation

**Status:** Planned / partially explored

---

### Phase 6 — Audio Planning

- Define the intended sound identity of the game
- Plan music and sound effect categories
- Identify where audio supports emotion, tension, and atmosphere
- Prepare for later implementation and testing

**Status:** Planned

---

### Phase 7 — Prototype Build

- Implement a playable internal prototype
- Test branching flow and route triggers
- Validate progression pacing
- Test readability, UX clarity, and basic game flow
- Replace temporary structures with cleaner systems where needed

**Status:** Future milestone

---

### Phase 8 — Vertical Slice / Demo Preparation

- Prepare a polished small-scale section of the game
- Refine presentation quality
- Replace selected placeholders
- Test player experience and first impressions
- Prepare public-facing materials where appropriate

**Status:** Future milestone

---

## Ongoing Priorities

These priorities may continue across multiple phases:

- Writing refinement
- Lore consistency checks
- Character voice consistency
- Scope control
- Asset organization
- Documentation cleanup
- Technical learning and experimentation

## Notes

This roadmap is a living document and will change as the project grows.

Some items may move earlier or later depending on:
- writing needs
- technical constraints
- art pipeline decisions
- available time
- scope adjustments

The goal is steady progress, not fake productivity theater.
